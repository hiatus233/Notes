# 游戏策划案：意志的远征 (The Willpower Expedition) —— 光之桥

主题： 桥 / 脆弱的连接 / 罕见病患者的内心世界

类型： 2D横版卷轴策略推塔 + 轻度Roguelite成长

视角： 横版侧视 (Side-Scrolling)

参考游戏： 核心战斗借鉴《死亡突围》，资源出兵借鉴《皇室战争》，局内成长借鉴《吸血鬼幸存者》，《冒险者的rogue之旅》

## 一、 游戏概述

本作构建了一个罕见病患者深层的内心世界。最初，这里因疾病的阴霾而失去色彩，是一片静止的灰暗虚空。玩家的核心使命是守护象征患者生命火花与希望的“微光核心（载具）”。

游戏的主要体验是“守护微光，点亮世界”。玩家需要利用随时间积攒的“希望之力”，召唤内心的美好具象——守护灵伙伴。他们将驱散前方象征沉重与阻碍的“灰雾实体”，在虚空中一步步铺就一条通往未知彼岸、长满鲜花与水晶的“光之桥”。整体基调追求吉卜力动画或《光遇》那种温暖、治愈且充满希望的正能量感。

参考分辨率：1920x1080p  实际分辨率：1920x1080p

## 二、 核心玩法循环

游戏的核心流程是一个不断推进防线的动态过程：

1. **推进与铺路：** 主角载具（微光核心）会持续、缓慢地向右侧移动。一个关键机制是，它经过的地方，脚下的灰暗虚空会被实时点亮，生成光之桥面。
    
2. **资源积累：** 战斗资源“希望之力”会随着时间流逝自动恢复。
    
3. **召唤决策：** 玩家消耗积攒的希望之力，在载具前方 strategic 地召唤不同功能的守护灵伙伴加入战场。
    
4. **对抗阻碍：** 守护灵被召唤后会自动向右挺进，遭遇灰雾实体（敌人）时进行战斗净化，清理前进道路上的路障。
    
5. **核心防守：** 玩家必须防止漏网的灰雾实体触碰到载具。一旦载具的光芒完全熄灭（血量归零），则判定为游戏失败。
    
6. **局内成长：** 游戏中加入了类幸存者的成长机制。每坚持一定时间或击败特定波次后，游戏会暂停并提供一个“三选一”的随机强化选项，提升本次冒险的能力。
    
7. **胜利推进：** 成功护送载具抵达关卡终点，即可解锁新的剧情篇章，并获得新的伙伴加入队伍。

## 三、 功能需求细案 (程序方向)
### 局内战斗系统

#### 移动距离
游戏中需要统计卷轴滚动的距离，抵达一定固定距离后会触发对应事件(剧情，敌人，boss等)
####  核心单位定义

##### 微光核心
微光核心是游戏中我们需要保护的主体对象。**无法移动**。
具有以下属性：

| 属性              | 说明            |
| --------------- | ------------- |
| 最大生命值 HP        | 最大生命值         |
| 当前生命值 CurrentHp | 当当前生命值归零时游戏失败 |
| 攻击力             | 受到敌人攻击后反弹的伤害  |

游戏开发初期采用球体来表示。以下是战斗原型图：
![[战斗房间图.png]]

##### 伙伴
是玩家消耗能量资源召唤出来的单位。
###### 生成机制
通过点击伙伴区召唤对应伙伴。
###### 特性
不同伙伴具有不同的特性，现阶段先考虑两种。
原型阶段战士类型的精灵图为为**正方体**，射手为**三角形**。友方战士为**绿色**，射手为**黄色**。

| 伙伴类型 | 说明                               |
| ---- | -------------------------------- |
| 战士   | 阻挡敌人前进，并在碰撞时对敌人造成伤害              |
| 射手   | 发动远程攻击，在子弹碰到敌人的时候对敌人造成伤害，不产生击退效果 |

##### 敌人

是玩家需要阻挡和击杀的单位。
###### 生成机制
从屏幕右侧的传送门中产生，按照每**固定时间段**内生成对应波次的敌人。
类型和精灵图同伙伴类型一致，需要做区分的是敌人均为**红色**。
以下是伙伴/敌人的属性表

| 属性               | 详细说明               |
| ---------------- | ------------------ |
| 最大生命值(HP)        | 最大承受伤害的能力          |
| 当前生命值(CurrentHP) | 当前能承受多少伤害，归零时死亡    |
| 攻击力(Attack)      | 每次攻击造成的伤害数值        |
| 防御力(Defense)     | 每次承受攻击减少的数值        |
| 速度(Speed)        | 决定移动速度(从左往右或是从右往左) |
| 攻击范围(Range)      | 能攻击到敌人的范围          |
|                  |                    |
伤害计算公式：

$$
当前生命值=当前生命值-|单次受到的攻击-自身防御力|
$$

#### 后退机制
当单位受到伤害之后时，会根据这次碰撞受到的伤害往移动方向的反方向产生一段位移。
具体计算规则如下：

$$
位移距离=受到的实际伤害*位移系数
$$

#### 迷失机制
当双方单位离开屏幕(摄像机规定区域)之后，默认死亡(他们在末世中迷失了)
我方单位伙伴迷失之后会返还**50**%的消耗资源量。


_(注：法术/卡牌系统作为后续优先级功能，视核心玩法开发进度决定是否加入。)_

#### 关卡生成

 地图是横向卷轴形式，地面会不断向前延伸。核心前方始终是未知的虚空，而后方则是它走过生成的实体路面。 
#### 局内成长
每生存60秒或击败精英怪后，游戏暂停，弹出一个界面给予3个随机增益选项（如：伙伴攻击提升、资源回复加快、载具回血等），玩家选择一项后游戏继续。

#### 胜利目标
当核心经过了一段固定**移动距离**会出现Boss，击败Boss赢得游戏胜利。
#### 剧情与伙伴系统

- **叙事结构：** 游戏按**移动距离**划分关卡。叙事应尽量碎片化、轻量化，通过关卡前后的简短对话或插画，讲述患者解开内心结的小故事。
    
- **伙伴解锁：** 初始玩家仅拥有最基础的伙伴。新的守护灵（如后续加入的远程单位）将作为通关特定大关卡的剧情奖励解锁，加入玩家的召唤列表。

## 四、 美术与氛围需求 (美术方向)

整体美术基调的核心在于“对比”：用温暖、梦幻、高饱和度的暖色调（代表希望）去对抗低饱和度的灰色调（代表疾病与阻碍）。

### 1. 场景表现：光之桥

战斗场景分辨率：
1920x1080p

场景是本作最重要的“题眼”。我们需要采用**视差滚动背景**来营造空间感。画面右侧那些玩家尚未抵达的区域，应该是一片灰蒙蒙的、静止的虚空，能看到远处模糊的灰色城市轮廓。


核心机制“铺路”需要强烈的视觉反馈：载具前方是断裂的灰色虚空，随着其推进，脚下要**实时生长**出一条发光的道路。路面上会自动长出水晶、发光的花草。主角走过的地方，世界瞬间恢复生机和色彩。

### 2. 角色与单位形象

载具 (微光核心)：

它是全场最暖的视觉焦点，必须看起来极其脆弱且珍贵。形象上可以想象为一个散发着暖金色光芒的小小剪影（像个被守护的孩子），坐在一个由发光藤蔓和水晶编织成的浮动平台上。

关于受击反馈，绝对不能表现流血或破碎。当它被触碰时，光芒应该不稳定地闪烁、变暗，周围的光晕缩小，让人产生心疼的感觉。

我方单位 (守护灵伙伴)：

他们是美好愿望和勇气的化身，自带柔光效果。色彩上采用金、橙、翠绿、天蓝等高饱和度暖色。

形象上不需要太具体的人形，可以是发光的玩具锡兵（肉盾定位），或者飘浮在空中用画笔发射颜料光弹的小精灵（射手定位）。

他们的攻击特效不是造成伤害，而是“净化”和“上色”，击中灰雾时应爆出彩色的光点。

敌方单位 (灰雾实体)：

设计原则是去恐怖化、去暴力化。它们代表的是阻碍和沉重情绪，而非邪恶。色彩以各种深浅的灰、浊色为主。

杂兵可以是缓慢蠕动的、没有固定形状的灰色烟尘团；精英怪可以是看起来很沉重的、巨大的生锈齿轮障碍物或枯萎荆棘球。

被击败时，它们不是死亡，而是“被净化”。视觉上应从灰色变为白色或彩色，然后化作一阵光点或几只彩色蝴蝶消散在空中。

### 3. UI 界面

整体 UI 风格需要轻盈、透亮，带有梦幻感。

血量条设计为一个装着金色光芒的容器槽，满能量时灿烂，低能量时黯淡，并带有柔和的呼吸动效。

资源条图标可以用一颗正在发芽的种子或星星来表示。

三选一的成长界面，希望做出像翻开命运塔罗牌一样的神圣仪式感。
