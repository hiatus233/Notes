# 一句话的核心玩法
玩家通过投掷初始的骰子撞击其他目标骰子，在回合结束时结算每一个动过的骰子的上截面，从而自动出对应数量的对应建筑，这些建筑每回合会召唤一定量的士兵与敌人进行对战。


玩家的主要构筑方面来源于携带的骰子，不同羁绊的骰子对应携带的建筑也不同，而玩家有一个初始白骰子，通过右下角的3D子场景中鼠标投掷初始白骰子来碰撞其他骰子，白骰子带有初始兵种，白骰子以及碰撞到的其他骰子翻出的点数代表这一轮出的兵种以及对应的兵种数量

掷骰子机制：初始阶段玩家可以移动鼠标提供一个预览效果，理想路径是一个直线上下浮动，加入一定的随机偏置。在按下鼠标左键之后发射骰子撞击棋盘上的其他骰子，每轮的结算是结算所有本轮动过的骰子的上截面数字。根据数字不同骰子有不同的效果。
初步分为兵种骰子，魔法骰子。
兵种骰子用于出兵，魔法骰子则是一些buff，伤害效果。


初始配有一个骰子卡组(初始白骰子+4个近战兵骰子+3个远程兵骰子)，也就是8个骰子的数量。

初始白骰子没有额外的效果，携带一个初始兵种，出的兵种数量即为基础骰子的1-6。

敌人的战斗形式也为出兵，不过意图会显示在上方，单次出兵存在一个对战时长，玩家可以选择1x,2x,3x。当时间耗尽则进入下一轮的出兵当中。

具体玩法为玩家的白骰子会以直线的形式射出，碰到的第一个骰子会在周围一定范围内寻找相同点数的骰子进行合成，如1和1合成了2点数的骰子，而这个合成出来的2点数的骰子会继续在一定范围内寻找相同点数的骰子进行合成，直到停止合成为止，而停止后进行结算时所有参与了合成的骰子的点数都会参与结算，不同类型的骰子会分别进行结算，如进攻类的骰子一起进行一个结算，防御类的骰子一起进行一个结算，最终根据相应的点数生成对应的建筑(这些建筑会分级别和数量)，且随着回合保留，你可以选择当回合结算这个类型并获取对应的建筑又或者积累去获取更好的建筑，这些建筑会有不同的效果，比如兵营，防御塔，铁匠铺等等

## 🎮 一、游戏概述

### 1.1 游戏简介

《骰子防卫战：智械危机》是一款将骰子合成Puzzle、网格建造策略和回合制战斗深度融合的创新型Roguelike游戏。

玩家扮演最后的防线指挥官，面对无尽的智械军团入侵。通过投掷骰子触发合成链，累积多维度标签点数，在n×n网格上建造建筑，最终与敌军进行回合制数值对抗。

### 1.2 核心特色

**🎲 物理合成链**

- 一次投掷引发连锁反应
- 策略与运气交织
- 长链合成带来高额奖励

**🏷️ 多维标签系统**

- 5种基础标签 + 7种元素标签 + 5种稀有标签
- 无限扩展性
- 流派组合深度极高

**🏗️ 网格建造策略**

- n x n网格布局
- 建筑位置影响协同效果
- 空间规划成为策略核心

**⚔️ 回合制战斗**

- 自动结算，无需微操
- 战前预测系统
- 专注于数值规划

**📈 复利成长**

- 建筑永久保留
- 协同效果叠加
- 越战越强的滚雪球体验

### 1.3 游戏口号

> **"一掷定乾坤，布阵铸帝国"**

### 1.4 设计哲学

**简化操作，深化策略**

- 去除实时塔防的微操压力
- 用回合制让玩家专注于规划
- 透明的数值让决策更有意义

**组合爆炸创造深度**

- 不依赖复杂规则
- 通过简单元素的组合产生复杂性
- 每局都有新发现

**尊重玩家时间**

- 单局可控在20-40分钟
- 可随时暂停保存
- 失败后也有收获（星尘货币）

```
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│ 阶段A: 玩家攻击阶段                         │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                             │
│ 1. 计算总攻击力                             │
│    公式:                                    │
│    总攻击 = Σ(每个建筑攻击力 × 协同倍率)    │
│                                             │
│    示例计算:                                │
│    建筑列表:                                │
│    • 兵营A: 50攻击 (相邻熔炉 ×2) = 100      │
│    • 兵营B: 50攻击 (相邻熔炉 ×2) = 100      │
│    • 兵营C: 50攻击 (无加成) = 50            │
│    • 箭塔A: 20攻击 (同行3建筑 +50%) = 30    │
│    总攻击力 = 100+100+50+30 = 280           │
│                                             │
│ 2. 应用特殊效果                             │
│    • 先手攻击: 第1回合伤害×1.5              │
│    • 暴击: 随机20%概率伤害×2                │
│    • 连锁: 伤害分散到多个敌人               │
│    • 元素效果: 附加额外伤害                 │
│                                             │
│ 3. 造成伤害                                 │
│    敌人血量 -= 实际伤害                     │
│                                             │
│    视觉演出:                                │
│    • 建筑依次发光                           │
│    • 发射攻击特效（根据类型）               │
│    • 敌人受击动画                           │
│    • 伤害数字飘出（大号字体）               │
│                                             │
│ 4. 检查敌人存活                             │
│    if (敌人血量 <= 0) {                     │
│      战斗胜利 → 跳转到战斗结果              │
│    }                                        │
│                                             │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│ 阶段B: 敌人攻击阶段                         │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                             │
│ 1. 计算敌人攻击力                           │
│    基础攻击 = 敌人攻击力                    │
│                                             │
│    应用Buff/Debuff:                         │
│    • 减速效果: 攻击力×0.7                   │
│    • 破甲效果: 攻击力×1.3                   │
│    • 狂暴效果: 攻击力×2                     │
│                                             │
│    示例:                                    │
│    敌人基础攻击: 200                        │
│    你的减速效果: -30%                       │
│    实际攻击 = 200 × 0.7 = 140               │
│                                             │
│ 2. 护盾抵消                                 │
│    总防御力 = Σ(每个建筑防御力 × 协同倍率)  │
│                                             │
│    示例:                                    │
│    总防御力: 150                            │
│    敌人攻击: 140                            │
│                                             │
│    if (敌人攻击 <= 总防御力) {              │
│      护盾完全抵消                           │
│      剩余伤害 = 0                           │
│      护盾值 -= 敌人攻击                     │
│    } else {                                 │
│      护盾被击破                             │
│      剩余伤害 = 敌人攻击 - 总防御力         │
│      护盾值 = 0                             │
│    }                                        │
│                                             │
│ 3. 主堡扣血                                 │
│    if (剩余伤害 > 0) {                      │
│      主堡血量 -= 剩余伤害                   │
│      播放主堡受损动画                       │
│      震屏特效                               │
│    }                                        │
│                                             │
│    视觉演出:                                │
│    • 敌人攻击动画                           │
│    • 护盾特效（如果抵消）                   │
│    • 护盾破裂动画（如果击破）               │
│    • 主堡血条减少                           │
│    • 主堡冒烟/裂纹（血量低时）              │
│                                             │
│ 4. 检查主堡存活                             │
│    if (主堡血量 <= 0) {                     │
│      战斗失败 → 跳转到游戏结束              │
│    }                                        │
│                                             │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│ 阶段C: 效果结算阶段                         │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                             │
│ 1. DOT效果结算                              │
│    • 燃烧: 敌人每回合 -X 血量                │
│    • 中毒: 敌人每回合 -Y 血量                │
│    • 腐蚀: 敌人防御 -Z%                      │
│                                             │
│ 2. 回复效果结算                             │
│    • 神圣: 主堡每回合 +X 血量                │
│    • 吸血: 转化伤害为护盾                   │
│    • 再生: 护盾每回合 +Y%                   │
│                                             │
│ 3. 控制效果结算                             │
│    • 冻结: 持续时间 -1                      │
│    • 减速: 持续时间 -1                      │
│    • 眩晕: 持续时间 -1                      │
│                                             │
│ 4. 护盾自然回复                             │
│    护盾值 += 总防御力 × 20%                 │
│    (最大不超过总防御力)                     │
│                                             │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                             │
│ 检查战斗是否结束:                           │
│ • 敌人死亡 → 胜利                           │
│ • 主堡死亡 → 失败                           │
│ • 双方存活 → 进入下一战斗回合              │
│                                             │
│ 玩家控制选项:                               │
│ [正常速度] [2倍速] [跳过动画]              │
└─────────────────────────────────────────────┘
                   ↓
             战斗结束判定
                   ↓
         ┌─────────┴─────────┐
         ↓                   ↓
    🎉 胜利！             💀 失败！
         │                   │
    显示战报              游戏结束
         │                   │
    敌人强度提升          显示结算
    (回合N+1)                 │
         │              解锁成就
         │              获得星尘
         │                   │
         └───────────────────→ 返回主菜单
```

## 🏷️ 三、标签系统

### 3.1 系统架构

标签系统是游戏的**解锁机制核心**，决定了玩家可以建造哪些建筑。

#### 3.1.1 双货币机制

游戏中有两种货币：

**货币1: 标签点数 (Tag Points)**

```yaml
定义:
  用于解锁建筑蓝图的永久累积资源

特性:
  • 永久累积，不会消耗
  • 每种标签独立计算
  • 达到阈值时解锁对应建筑
  • 显示为进度条形式

作用:
  "钥匙" - 决定你能建造什么

获得途径:
  • 合成骰子（主要来源）
  • 特殊事件
  • 成就奖励

示例:
  当前状态:
  🗡️ 进攻: 22点 → 已解锁[基础兵营][烈焰兵营]
  🔥 火焰: 25点 → 已解锁[烈焰兵营][熔炉]
  
  新建筑解锁条件:
  [巨龙巢穴] 需要: 进攻≥50, 火焰≥40
  还差: 进攻28点, 火焰15点
```

**货币2: 建造点 (Build Points)**

```yaml
定义:
  用于实际建造/升级建筑的消耗性货币

特性:
  • 每回合获得，每回合消耗
  • 可以积累，但通常会用完
  • 显示为货币数字

作用:
  "钱" - 决定你能建造多少

获得途径:
  每回合基础: 10点
  合成链奖励:
    2次链: +5点
    3次链: +10点
    4次链: +15点
    5+次链: +25点
  
  经济建筑:
    [采石场Lv1]: +3点/回合
    [采石场Lv2]: +6点/回合
    [采石场Lv3]: +9点/回合
    [高级采石场]: +15点/回合
    [世界树]: +15点/回合
  
  战斗奖励:
    完美防守: +5点
    速战速决: +10点

消耗用途:
  • 建造新建筑 (8-120点)
  • 升级建筑 (建造成本×2)
  • 特殊能力 (部分建筑的主动技能)

示例:
  回合开始: 10点 (基础)
  投掷合成: +15点 (3次链)
  经济建筑: +6点 (2个采石场)
  当前总计: 31点
  
  建造选择:
  • 2个[基础兵营] = 16点, 剩余15点
  • 1个[烈焰兵营] = 15点, 剩余16点
  • 1个[采石场] = 12点, 剩余19点
```

#### 3.1.2 为什么要双货币？

```
单一货币的问题:
  如果只有"点数"，玩家攒够100点就直接买最强建筑
  → 没有策略选择
  → 每局玩法雷同

双货币的优势:
  标签系统负责"解锁"
  建造点负责"限制"
  
  结果:
  • 玩家可能解锁了10种建筑
  • 但建造点只够建2-3个
  • 必须做出取舍
  • 每回合都有策略深度
  
  长期演变:
  前期: 标签少，建筑选择少
  中期: 标签多，选择丰富，但建造点有限
  后期: 有经济建筑，建造点充裕，开始大规模建造
```

### 3.2 标签分类体系

标签分为三个层级，形成递进的解锁树：

#### 3.2.1 基础标签（一级标签）

这是游戏的**骨架**，对应核心游戏维度。

|标签|图标|定位|对应建筑产出|战斗作用|
|---|---|---|---|---|
|🗡️ 进攻|Attack|输出系统|攻击力建筑|造成伤害|
|🛡️ 防御|Defense|生存系统|防御力建筑|抵消伤害|
|💰 经济|Economy|资源系统|建造点生产|加速发展|
|⚡ 速度|Speed|节奏系统|攻速/先手|抢先攻击|
|🌀 控制|Control|辅助系统|减益效果|削弱敌人|

**设计原则：**

- 每个基础标签对应一个核心游戏维度
- 所有建筑至少需要1个基础标签
- 基础标签永远有用，不会过时

**获取难度：**

- 最容易获得
- 普通骰子主要提供基础标签
- 倍率通常为 1.0x

#### 3.2.2 元素标签（二级标签）

这是游戏的**血肉**，提供风味和特色机制。

|标签|图标|效果特色|战斗表现|常见组合|
|---|---|---|---|---|
|🔥 火焰|Fire|持续燃烧|每回合造成DOT伤害|进攻+火焰|
|❄️ 冰霜|Frost|减速冻结|降低敌人攻击力|控制+冰霜|
|⚡ 闪电|Lightning|连锁攻击|伤害分散到多个敌人|速度+闪电|
|☠️ 毒素|Poison|腐蚀削弱|降低敌人防御|控制+毒素|
|✨ 神圣|Holy|治疗净化|回复主堡血量|防御+神圣|
|🌑 黑暗|Dark|吸血诅咒|伤害转化为护盾|进攻+黑暗|
|🌿 自然|Nature|成长再生|建筑效果逐渐提升|经济+自然|

**设计原则：**

- 元素标签提供特殊战斗机制
- 可以跨基础标签组合
- 创造build流派

**战斗效果详解：**

```yaml
🔥 火焰效果:
  类型: DOT (Damage Over Time)
  机制: 
    • 攻击命中后，敌人获得"燃烧"状态
    • 每个战斗回合开始，敌人受到X点伤害
    • 持续3-5个战斗回合
    • 多个火焰建筑的燃烧效果叠加
  
  示例:
    建筑A: 燃烧10/回合
    建筑B: 燃烧10/回合
    结果: 敌人每回合受到20点燃烧伤害

❄️ 冰霜效果:
  类型: 减速 (Debuff)
  机制:
    • 攻击命中后，敌人获得"减速"状态
    • 敌人攻击力降低X%
    • 持续2-3个战斗回合
    • 多个冰霜建筑效果叠加
  
  示例:
    建筑A: 减速20%
    建筑B: 减速20%
    结果: 敌人攻击力-40%

⚡ 闪电效果:
  类型: 连锁伤害
  机制:
    • 攻击不针对单个敌人
    • 伤害分散到多个敌人身上
    • 总伤害×1.5（补偿分散的损失）
  
  示例:
    建筑攻击力: 100
    敌人数量: 3个
    结果: 每个敌人受到50伤害 (100×1.5÷3)

☠️ 毒素效果:
  类型: 破甲 (Debuff)
  机制:
    • 攻击命中后，敌人获得"中毒"状态
    • 敌人防御力降低X%
    • 后续所有伤害提升
  
  示例:
    敌人护甲: 30%减伤
    毒素效果: -20%护甲
    结果: 敌人护甲变为10%减伤

✨ 神圣效果:
  类型: 回复 (Heal)
  机制:
    • 每个战斗回合结束时
    • 主堡回复X点血量
    • 多个神圣建筑效果叠加
  
  示例:
    建筑A: 回复5/回合
    建筑B: 回复5/回合
    结果: 主堡每回合回复10血量

🌑 黑暗效果:
  类型: 吸血 (Lifesteal)
  机制:
    • 造成伤害的X%转化为护盾
    • 立即增加护盾值
    • 可以超过最大护盾值
  
  示例:
    造成伤害: 200
    吸血比例: 20%
    结果: 护盾+40

🌿 自然效果:
  类型: 成长 (Growth)
  机制:
    • 建筑效果每回合提升X%
    • 永久叠加
    • 适合长期战斗
  
  示例:
    建筑基础攻击: 50
    成长速度: 5%/回合
    第1回合: 50
    第2回合: 52.5
    第3回合: 55.1
    第10回合: 81.4
```

**获取难度：**

- 中等难度
- 稀有骰子主要提供元素标签
- 倍率通常为 0.5-0.8x

#### 3.2.3 稀有标签（三级标签）

这是游戏的**灵魂**，晚期才能解锁的超级能力。

|标签|图标|解锁条件|战斗效果|
|---|---|---|---|
|🎭 混沌|Chaos|火焰≥20 + 冰霜≥20|随机效果，高风险高回报|
|⏰ 时间|Time|速度≥30 + 控制≥30|时停，敌人跳过攻击回合|
|🌌 空间|Space|防御≥40 + 控制≥20|折射，反弹伤害|
|💚 生命|Life|自然≥30 + 神圣≥30|复活，死亡后重生|
|💀 死亡|Death|黑暗≥30 + 毒素≥30|献祭，用血换伤害|

**设计原则：**

- 需要组合多个标签才能解锁
- 提供游戏改变级的能力
- 给hardcore玩家追求的目标

**效果详解：**

```yaml
🎭 混沌效果:
  解锁: 火焰≥20, 冰霜≥20
  机制:
    • 每个战斗回合开始时
    • 随机触发以下效果之一:
      - 攻击力×3（33%概率）
      - 防御力×3（33%概率）
      - 敌人血量-50%（33%概率）
    • 不可控，但强度极高
  
  适合: 赌徒型玩家

⏰ 时间效果:
  解锁: 速度≥30, 控制≥30
  机制:
    • 时停: 每3个战斗回合
    • 敌人跳过1次攻击
    • 你可以连续攻击2次
  
  示例:
    第1回合: 正常战斗
    第2回合: 正常战斗
    第3回合: 时停触发
      • 你攻击2次
      • 敌人攻击0次
    第4回合: 正常战斗

🌌 空间效果:
  解锁: 防御≥40, 控制≥20
  机制:
    • 折射: 你承受的伤害的X%
    • 反弹给敌人
    • 相当于同时防御和攻击
  
  示例:
    敌人攻击: 200
    你的护盾: 150
    剩余伤害: 50
    折射比例: 30%
    结果: 
      • 你的主堡-50血
      • 敌人-15血 (50×30%)

💚 生命效果:
  解锁: 自然≥30, 神圣≥30
  机制:
    • 复活: 主堡血量归零时
    • 立即复活至30%最大生命
    • 整局游戏只能触发1次
  
  示例:
    主堡最大血量: 100
    主堡当前血量: 5
    敌人攻击: 20
    结果:
      • 主堡血量本应变为-15（死亡）
      • 复活触发
      • 主堡血量变为30
      • 战斗继续

💀 死亡效果:
```

### 4.3 骰子桌面生成规则

每回合开始时，桌面会生成一批骰子供玩家投掷。

#### 4.3.1 生成算法

```python
def generate_dice_table(round_number):
    """
    生成当前回合的骰子桌面
    
    参数:
        round_number: 当前回合数
    
    返回:
        dice_list: 骰子列表
    """
    
    # 1. 确定骰子数量
    if round_number <= 3:
        dice_count = 12
    elif round_number <= 7:
        dice_count = 15
    elif round_number <= 12:
        dice_count = 18
    else:
        dice_count = 20
    
    # 2. 确定稀有度分布
    rarity_pool = []
    
    if round_number <= 3:
        # 新手阶段：主要是普通骰子
        rarity_pool = [
            ("common", 0.80),   # 80%普通
            ("rare", 0.20),     # 20%稀有
        ]
    elif round_number <= 7:
        # 成长阶段：引入史诗
        rarity_pool = [
            ("common", 0.50),
            ("rare", 0.40),
            ("epic", 0.10),
        ]
    elif round_number <= 12:
        # 挑战阶段：更多高级骰子
        rarity_pool = [
            ("common", 0.30),
            ("rare", 0.40),
            ("epic", 0.25),
            ("legendary", 0.05),
        ]
    else:
        # 后期：极限挑战
        rarity_pool = [
            ("rare", 0.40),
            ("epic", 0.40),
            ("legendary", 0.20),
        ]
    
    # 3. 生成骰子列表
    dice_list = []
    for i in range(dice_count):
        # 根据权重随机选择稀有度
        rarity = weighted_random(rarity_pool)
        
        # 从对应稀有度池中随机选择骰子
        dice = select_random_dice(rarity, round_number)
        
        # 随机点数（1-6）
        dice.value = random.randint(1, 6)
        
        dice_list.append(dice)
    
    # 4. 布局策略
    # 保证有一定数量的"可合成对"
    ensure_mergeable_pairs(dice_list, min_pairs=3)
    
    # 5. 位置分布
    positions = generate_random_positions(dice_count)
    for dice, pos in zip(dice_list, positions):
        dice.position = pos
    
    return dice_list


def ensure_mergeable_pairs(dice_list, min_pairs):
    """
    确保桌面上有足够的可合成对
    
    逻辑:
    - 至少有min_pairs组相同点数的骰子聚在一起
    - 避免玩家完全无法合成的情况
    """
    # 统计每个点数的骰子
    value_groups = {}
    for dice in dice_list:
        if dice.value not in value_groups:
            value_groups[dice.value] = []
        value_groups[dice.value].append(dice)
    
    # 找出单独的骰子
    lonely_dice = []
    for value, group in value_groups.items():
        if len(group) == 1:
            lonely_dice.extend(group)
    
    # 将一些单独的骰子改为相同点数
    pairs_created = 0
    for i in range(0, len(lonely_dice) - 1, 2):
        if pairs_created >= min_pairs:
            break
        
        # 让这两个骰子点数相同
        dice1 = lonely_dice[i]
        dice2 = lonely_dice[i + 1]
        dice2.value = dice1.value
        
        # 并且放在相邻位置
        place_adjacent(dice1, dice2)
        
        pairs_created += 1
```

#### 5.3.1 协同类型


```python
def initialize_battle(player_buildings, enemy_data, main_base_hp):
    """
    初始化战斗
    """
    
    # 玩家方数据
    player = {
        "total_attack": calculate_total_attack(player_buildings),
        "total_defense": calculate_total_defense(player_buildings),
        "special_effects": collect_special_effects(player_buildings),
        "synergies": collect_synergies(player_buildings),
        "main_base_hp": main_base_hp,
        "max_main_base_hp": 100,
        "shield": calculate_total_defense(player_buildings)
    }
    
    # 敌人方数据
    enemy = {
        "max_hp": enemy_data["hp"],
        "current_hp": enemy_data["hp"],
        "attack": enemy_data["attack"],
        "armor": enemy_data.get("armor", 0),
        "special_abilities": enemy_data.get("special", []),
        "debuffs": []
    }
    
    return player, enemy


def calculate_total_attack(buildings):
    """
    计算总攻击力
    """
    total = 0
    
    for building in buildings:
        # 基础攻击
        base_attack = building.stats.attack
        
        # 协同倍率
        synergy_multiplier = calculate_synergy_multiplier(
            building, 
            buildings
        )
        
        # 最终攻击
        final_attack = base_attack * synergy_multiplier
        total += final_attack
    
    return total


def calculate_total_defense(buildings):
    """
    计算总防御力(护盾)
    """
    total = 0
    
    for building in buildings:
        base_defense = building.stats.defense
        synergy_multiplier = calculate_synergy_multiplier(
            building,
            buildings
        )
        final_defense = base_defense * synergy_multiplier
        total += final_defense
    
    return total
```

#### 6.1.2 战斗回合流程

```python
def battle_round(player, enemy, round_number):
    """
    执行一个战斗回合
    
    返回: 
        - "player_win": 玩家胜利
        - "enemy_win": 敌人胜利
        - "continue": 继续战斗
    """
    
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 阶段1: 玩家攻击阶段
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    # 1.1 计算攻击力
    attack = player["total_attack"]
    
    # 1.2 应用先手效果(第1回合)
    if round_number == 1 and has_first_strike(player):
        attack *= get_first_strike_multiplier(player)
        log_combat("⚡ 先手攻击! 伤害×1.5")
    
    # 1.3 应用DOT效果
    dot_damage = calculate_dot_damage(player, enemy)
    
    # 1.4 应用敌人护甲
    armor_reduction = 1 - (enemy["armor"] / 100)
    actual_damage = attack * armor_reduction
    
    # 1.5 造成伤害
    enemy["current_hp"] -= actual_damage
    log_combat(f"🗡️ 造成 {actual_damage} 伤害")
    
    # 1.6 应用DOT
    enemy["current_hp"] -= dot_damage
    log_combat(f"🔥 燃烧/中毒造成 {dot_damage} 伤害")
    
    # 1.7 检查敌人存活
    if enemy["current_hp"] <= 0:
        log_combat("💀 敌人被击杀!")
        return "player_win"
    
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 阶段2: 敌人攻击阶段
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    # 2.1 计算敌人攻击力
    enemy_attack = enemy["attack"]
    
    # 2.2 应用玩家的debuff
    debuff_multiplier = calculate_debuff_multiplier(player, enemy)
    enemy_attack *= debuff_multiplier
    
    if debuff_multiplier < 1.0:
        reduction = (1 - debuff_multiplier) * 100
        log_combat(f"❄️ 减速/冻结效果: 敌人攻击力-{reduction}%")
    
    # 2.3 护盾抵消
    if enemy_attack <= player["shield"]:
        # 护盾完全抵消
        player["shield"] -= enemy_attack
        log_combat(f"🛡️ 护盾抵消 {enemy_attack} 伤害")
        log_combat(f"   剩余护盾: {player['shield']}")
    else:
        # 护盾被击破
        overkill = enemy_attack - player["shield"]
        log_combat(f"💥 护盾被击破! ({player['shield']})")
        player["shield"] = 0
        
        # 主堡扣血
        player["main_base_hp"] -= overkill
        log_combat(f"🏰 主堡受到 {overkill} 伤害")
        log_combat(f"   主堡血量: {player['main_base_hp']}/{player['max_main_base_hp']}")
    
    # 2.4 反伤效果
    if enemy_attack > 0:
        reflect_damage = calculate_reflect_damage(player, enemy_attack)
        if reflect_damage > 0:
            enemy["current_hp"] -= reflect_damage
            log_combat(f"🌌 折射反弹 {reflect_damage} 伤害给敌人")
    
    # 2.5 检查主堡存活
    if player["main_base_hp"] <= 0:
        # 检查复活效果
        if has_revive(player) and not player.get("revived", False):
            revive_hp = player["max_main_base_hp"] * 0.5
            player["main_base_hp"] = revive_hp
            player["revived"] = True
            log_combat(f"💚 复活触发! 主堡恢复至 {revive_hp} 血量")
        else:
            log_combat("💀 主堡被摧毁!")
            return "enemy_win"
    
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 阶段3: 效果结算阶段
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    # 3.1 回复效果
    heal_amount = calculate_heal(player)
    if heal_amount > 0:
        player["main_base_hp"] = min(
            player["main_base_hp"] + heal_amount,
            player["max_main_base_hp"]
        )
        log_combat(f"✨ 主堡回复 {heal_amount} 血量")
    
    # 3.2 吸血转护盾
    lifesteal_shield = calculate_lifesteal(player, actual_damage)
    if lifesteal_shield > 0:
        player["shield"] += lifesteal_shield
        log_combat(f"🌑 吸血转化 {lifesteal_shield} 护盾")
    
    # 3.3 护盾自然回复
    shield_regen = player["total_defense"] * 0.2
    player["shield"] = min(
        player["shield"] + shield_regen,
        player["total_defense"]
    )
    
    # 3.4 更新DOT持续时间
    update_dot_durations(enemy)
    
    # 3.5 更新debuff持续时间
    update_debuff_durations(enemy)
    
    # 战斗继续
    return "continue"
```

```python
        current_attack *= 2
```

```python
# 玩家成长速度模型

class PlayerGrowth:
    def __init__(self):
        self.buildings = []
        self.total_attack = 0
        self.total_defense = 0
    
    def calculate_growth(self, round_number):
        """
        计算玩家在第N回合的预期战力
        """
        
        # 1. 建造点收入
        build_points_per_round = self.calculate_build_points(round_number)
        
        # 2. 预期建筑数量
        total_build_points = build_points_per_round * round_number
        average_building_cost = 20  # 平均成本
        expected_buildings = total_build_points / average_building_cost
        
        # 3. 预期攻击力
        average_attack_per_building = 60  # 平均攻击(含协同)
        expected_attack = expected_buildings * average_attack_per_building
        
        # 4. 预期防御力
        average_defense_per_building = 40  # 平均防御(含协同)
        expected_defense = expected_buildings * average_defense_per_building
        
        return {
            "round": round_number,
            "build_points": total_build_points,
            "buildings": expected_buildings,
            "attack": expected_attack,
            "defense": expected_defense
        }
    
    def calculate_build_points(self, round_number):
        """
        计算每回合建造点收入
        """
        base = 10  # 基础
        merge_bonus = 10  # 平均合成链奖励
        economy_buildings = round_number * 0.3  # 预期经济建筑数量
        economy_income = economy_buildings * 8  # 平均每个+8/回合
        
        return base + merge_bonus + economy_income


# 平衡目标

def balance_check(round_number):
    """
    检查平衡性
    """
    player = PlayerGrowth()
    player_stats = player.calculate_growth(round_number)
    enemy_stats = calculate_enemy_stats(round_number)
    
    # 目标1: 玩家应该能在2-3回合内击杀敌人
    rounds_to_kill = enemy_stats["hp"] / player_stats["attack"]
    
    # 目标2: 玩家应该能承受2-3次敌人攻击
    hits_to_die = (100 + player_stats["defense"]) / enemy_stats["attack"]
    
    # 平衡判定
    is_balanced = (
        2 <= rounds_to_kill <= 3 and
        2 <= hits_to_die <= 3
    )
    
    return {
        "round": round_number,
        "rounds_to_kill": rounds_to_kill,
        "hits_to_die": hits_to_die,
        "balanced": is_balanced,
        "difficulty": "perfect" if is_balanced else ("too_easy" if rounds_to_kill < 2 else "too_hard")
    }
```
