骰子+自动出兵+肉鸽（核心还是卡组构筑）

美术风格：2D像素（骰子场景为3D后像素化处理）
主场景相机视角：俯视角 
# 一句话的核心玩法
玩家通过投掷初始的骰子撞击其他目标骰子，在回合结束时结算每一个动过的骰子的上截面，从而自动出对应数量的对应建筑，这些建筑每回合会召唤一定量的士兵与敌人进行对战。


玩家的主要构筑方面来源于携带的骰子，不同羁绊的骰子对应携带的建筑也不同，而玩家有一个初始白骰子，通过右下角的3D子场景中鼠标投掷初始白骰子来碰撞其他骰子，白骰子带有初始兵种，白骰子以及碰撞到的其他骰子翻出的点数代表这一轮出的兵种以及对应的兵种数量

掷骰子机制：初始阶段玩家可以移动鼠标提供一个预览效果，理想路径是一个直线上下浮动，加入一定的随机偏置。在按下鼠标左键之后发射骰子撞击棋盘上的其他骰子，每轮的结算是结算所有本轮动过的骰子的上截面数字。根据数字不同骰子有不同的效果。
初步分为兵种骰子，魔法骰子。
兵种骰子用于出兵，魔法骰子则是一些buff，伤害效果。


初始配有一个骰子卡组(初始白骰子+4个近战兵骰子+3个远程兵骰子)，也就是8个骰子的数量。

初始白骰子没有额外的效果，携带一个初始兵种，出的兵种数量即为基础骰子的1-6。

敌人的战斗形式也为出兵，不过意图会显示在上方，单次出兵存在一个对战时长，玩家可以选择1x,2x,3x。当时间耗尽则进入下一轮的出兵当中。

具体玩法为玩家的白骰子会以直线的形式射出，碰到的第一个骰子会在周围一定范围内寻找相同点数的骰子进行合成，如1和1合成了2点数的骰子，而这个合成出来的2点数的骰子会继续在一定范围内寻找相同点数的骰子进行合成，直到停止合成为止，而停止后进行结算时所有参与了合成的骰子的点数都会参与结算，不同类型的骰子会分别进行结算，如进攻类的骰子一起进行一个结算，防御类的骰子一起进行一个结算，最终根据相应的点数生成对应的建筑(这些建筑会分级别和数量)，且随着回合保留，你可以选择当回合结算这个类型并获取对应的建筑又或者积累去获取更好的建筑，这些建筑会有不同的效果，比如兵营，防御塔，铁匠铺等等



# 🎲 标签系统完整设计文档

## 一、核心机制设计

### 1.1 标签的本质

**标签不是资源，而是"能力维度"**

```
传统思维：
资源 → 货币 → 购买建筑

标签思维：
标签 → 解锁条件 → 激活建筑
```

**关键原则：**

- 玩家不"花费"标签点数，标签是**永久累积**的
- 标签达到阈值时，解锁对应建筑的**建造权**
- 建造建筑需要消耗"建造点"（通用货币）

---

### 1.2 双货币系统

```
系统架构：

标签点数（Tag Points）
├─ 作用：解锁建筑蓝图
├─ 来源：合成骰子
└─ 特性：永久累积，不消耗

建造点（Build Points）
├─ 作用：实际建造建筑
├─ 来源：每回合固定获得 + 经济标签加成
└─ 特性：消耗品，需要管理
```

**为什么要双货币？**

```
单一货币的问题：
玩家攒够100点 → 直接买最强建筑 → 没有策略选择

双货币的好处：
标签累积 → 解锁多个蓝图 → 但建造点有限 → 必须选择
```

**游戏循环示例：**

```
第1回合：
├─ 合成了6点 #进攻 骰子
├─ 累积标签：{进攻: 6}
├─ 解锁：[基础兵营] (需求 #进攻≥5)
├─ 获得建造点：10点
└─ 建造：[基础兵营] (花费 8建造点)

第2回合：
├─ 合成了8点 #火焰 骰子
├─ 累积标签：{进攻: 6, 火焰: 8}
├─ 解锁：[火焰兵营] (需求 #进攻≥5 + #火焰≥5)
├─ 获得建造点：10点
└─ 选择：升级现有兵营？还是建新的火焰兵营？
```

## 二、标签体系设计

### 2.1 基础标签（一级标签）

这是游戏的**骨架**，永远不变：

|标签|图标|定位|核心概念|
|---|---|---|---|
|#进攻|🗡️|输出系统|主动伤害|
|#防御|🛡️|生存系统|被动保护|
|#经济|💰|资源系统|加速发展|
|#速度|⚡|节奏系统|攻速/移速|
|#控制|🌀|辅助系统|减速/眩晕|

**设计原则：**

- 每个基础标签对应一个**游戏维度**
- 所有建筑都至少需要1个基础标签
- 基础标签永远有用（不会过时）

---

### 2.2 元素标签（二级标签）

这是游戏的**血肉**，提供风味和深度：

|标签|图标|效果特色|与基础标签的关系|
|---|---|---|---|
|#火焰|🔥|持续伤害(DOT)|常与 #进攻 搭配|
|#冰霜|❄️|减速/冻结|常与 #控制 搭配|
|#闪电|⚡|链式攻击|常与 #速度 搭配|
|#毒素|☠️|削弱敌人|常与 #进攻 搭配|
|#神圣|✨|治疗/buff|常与 #防御 搭配|
|#黑暗|🌑|吸血/诅咒|常与 #进攻 搭配|
|#自然|🌿|生长/再生|常与 #经济 搭配|

**设计原则：**

- 元素标签提供**特殊机制**
- 元素标签可以跨基础标签组合
- 元素标签创造build流派

---

### 2.3 稀有标签（三级标签）

这是游戏的**灵魂**，晚期才解锁：

| 标签  | 图标  | 解锁条件            | 效果          |

**设计原则：**

- 稀有标签需要**组合多个标签**才能解锁

- 给hardcore玩家追求的目标

## 五、合成系统详细设计

### 5.1 合成的三个阶段

#### 阶段1：瞄准与发射

```
玩家操作：
1. 点击并拖拽白色骰子
2. 看到瞄准线（类似《愤怒的小鸟》）
3. 看到"预测高亮"：
   ├─ 绿色: 必定击中的第一个骰子
   ├─ 黄色: 可能触发的合成范围
   └─ 橙色: 二次/三次合成的潜在目标
4. 松开鼠标，发射

视觉反馈：
- 骰子沿直线飞行（有拖尾特效）
- 击中第一个骰子时，震屏+音效
```

---

#### 阶段2：合成链计算

```
合成规则：
1. 白骰子击中第一个目标骰子（触发点）
2. 触发点周围3米范围内，搜索"相同点数"的骰子
3. 找到的骰子与触发点合成 → 点数+1
4. 新骰子继续搜索周围3米范围
5. 重复直到没有可合成的目标

示例：
桌面布局：
  A(3) - B(3) - C(3)
    \      |      /
     D(4) E(3) F(6)

你击中E(3)：
├─ 第1次合成: E(3) + A(3) + B(3) + C(3) = 新骰子(6)
├─ 新骰子(6)搜索范围内的6点骰子
├─ 找到F(6)
└─ 第2次合成: 新骰子(6) + F(6) = 最终骰子(12)

合成链: 2次
参与合成的骰子: 5个
最终点数: 12
```

**合成范围视觉化：**

```
每个骰子周围有虚线圆圈表示"搜索半径"
合成发生时：
├─ 被吸引的骰子飞向合成中心
├─ 合并时爆发光效
└─ 新骰子比旧骰子大20%
```

### 7.2 高级技巧

#### 技巧1：标签狙击

```
策略：
观察桌面骰子分布
如果看到大量某一元素骰子聚集
→ 立即调整build，专精该元素

示例：
桌面出现6个火焰骰子都是5-6点
→ 这回合全打火焰骰子
→ 一回合获得40+火焰标签
→ 直接解锁[凤凰祭坛]
```

---

#### 技巧2：延迟建造

```
策略：
不要每回合都建新建筑
有时候"攒"建造点更划算

示例：
当前有30建造点，可以建2个普通兵营
但我选择不建，等下回合攒到80建造点
→ 直接建[世界树]
→ 每回合+15建造点，长期收益更高
```

---

#### 技巧3：协同叠加

```
策略：
优先建有"协同效果"的建筑组合

示例：
建筑A: "每有1个火焰建筑，伤害+10%"
建筑B: "每有1个火焰建筑，攻速+5%"

如果我建3个火焰建筑：
├─ 建筑A和B各自触发3次
└─ 总增益: +30%伤害, +15%攻速

→ 1+1+1 > 3
```

---

#### 技巧4：敌人预判

```
观察敌人类型，调整build

敌人类型提示：
"下波敌人:大量低血快速单位"
→ 建[闪电法师塔](AOE)

"下波敌人:少量高血坦克"
→ 建[暗影刺客营](单体爆发)

"下波敌人:空军单位"
→ 建[防空箭塔]
```
