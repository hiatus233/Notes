# 一句话的核心玩法
玩家通过投掷初始的骰子撞击其他目标骰子，在回合结束时结算每一个动过的骰子的上截面，从而自动出对应数量的对应建筑，这些建筑每回合会召唤一定量的士兵与敌人进行对战。

玩家的主要构筑方面来源于携带的骰子，不同羁绊的骰子对应携带的建筑也不同，而玩家有一个初始白骰子，通过右下角的3D子场景中鼠标投掷初始白骰子来碰撞其他骰子，白骰子带有初始兵种，白骰子以及碰撞到的其他骰子翻出的点数代表这一轮出的兵种以及对应的兵种数量

掷骰子机制：初始阶段玩家可以移动鼠标提供一个预览效果，理想路径是一个直线上下浮动，加入一定的随机偏置。在按下鼠标左键之后发射骰子撞击棋盘上的其他骰子，每轮的结算是结算所有本轮动过的骰子的上截面数字。根据数字不同骰子有不同的效果。
初步分为兵种骰子，魔法骰子。
兵种骰子用于出兵，魔法骰子则是一些buff，伤害效果。
初始配有一个骰子卡组(初始白骰子+4个近战兵骰子+3个远程兵骰子)，也就是8个骰子的数量。
初始白骰子没有额外的效果，携带一个初始兵种，出的兵种数量即为基础骰子的1-6。
敌人的战斗形式也为出兵，不过意图会显示在上方，单次出兵存在一个对战时长，玩家可以选择1x,2x,3x。当时间耗尽则进入下一轮的出兵当中。

具体玩法为玩家的白骰子会以直线的形式射出，碰到的第一个骰子会在周围一定范围内寻找相同点数的骰子进行合成，如1和1合成了2点数的骰子，而这个合成出来的2点数的骰子会继续在一定范围内寻找相同点数的骰子进行合成，直到停止合成为止，而停止后进行结算时所有参与了合成的骰子的点数都会参与结算，不同类型的骰子会分别进行结算，如进攻类的骰子一起进行一个结算，防御类的骰子一起进行一个结算，最终根据相应的点数生成对应的建筑(这些建筑会分级别和数量)，且随着回合保留，你可以选择当回合结算这个类型并获取对应的建筑又或者积累去获取更好的建筑，这些建筑会有不同的效果，比如兵营，防御塔，铁匠铺等等
# 骰子
![[骰子.png]]
玩家每回合投掷初始骰子并撞击其他骰子以触发合成，参与合成的骰子的点数会被投入计算本回合获取的建筑点数。
## 骰子特性
点数：一般情况下一个骰子只有一个点数，每个面都会显示这个点数。
buff：具有多种buff类型(这里指的是附加特性，如负片等)，可用于叠加获得的建筑点数(buff参考小丑牌)。

# 建筑
玩家每回合投掷骰子结束后进入到放置建筑阶段，建筑的具体效果就是我们的攻击/防御/获取局外金币的手段
## 建筑点
建筑需要建筑点进行放置，建筑点通过合成骰子获得。
## 建筑占位
建筑占据的格子有不同类型，而不是单一的单格。

# 敌人
## AI逻辑
敌人也具有棋盘并会展示意图，目前有这么几个设计的方向：1.开局即放置了所有建筑
2.每回合也会建造建筑，类似于杀戮尖塔中有固定的AI行为逻辑 
3.采用类似恶魔牌里面的异步自走棋逻辑
4.真实模拟，设计一套算法来评估玩家行为并在下一回合做出对应对策
